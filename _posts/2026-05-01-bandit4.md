## 1. bandit 15->16 문제
풀이 : 이 문제는 openssl s_client를 어떻게 쓰는지를 아는 문제입니다.

openssl : SSL/TLS Protocol로 구현한 라이브러리 중 하나로, 통신전문과 문서를 암호화

s_clinet : SSL/TLS를 사용하는 원격 호스트에 접속하기 위한 명령어

사용법 : openssl s_client [option]
이번 문제에서 사용할 option은 '-connect host:post'이기때문에

## 3. bandit 17->18 문제
풀이 : 이번 문제는 *'이 레벨을 완료했는데 bandit18에 로그인하려고 할 때 'Byebye!'라는 메시지가 표시되면,
이는 다음 레벨인 bandit19와 관련이 있습니다.'* 라고 되어 있어서 접속했더니 바로 튕겨버렸습니다.
그래서 SCP를 이용해서 readme 파일을 읽어오면 되겠다 해서 SCP에 대해 검색을 했습니다.

먼저 file로 qr을 열면 POSIX tar archive가 나옵니다. 이걸 tar로 어떻게 쓰는가에 대해선
검색을 해서 압축 풀기 위한 tar를 검색했더니 mv qr->qr.tar 그러고 tar -xvf qr.tar해야 이 file 을 쓸 수 있다는걸 알 수 있었습니다.

Scp 사용법 : scp -P 2220 [사용자명]@[호스트]:[원격파일경로] [로컬저장경로]
 
<img src="assets/bandit/18.png" width="400" alt="Bandit 17-18 풀이 이미지">

**비밀번호** 가 나와서 bandit18로 넘어갔습니다.

## 3. bandit 19->20 문제
풀이 :  앞전 문제랑 같게 풀려고 했습니다. 일단 똑같이 빨강 블럭처리 되어있는걸 file로 읽어봤더니
suconnect 는 특정 포트로 접속을 시도하여 데이터를 주고받는 실행 파일이며, 실행 시 소유자(bandit21)의 권한을 빌려오는 기능을 가졌다는
것을 알 수 있습니다.

<img src="assets/bandit/20-1.png" width="400" alt="Bandit 19-20 풀이 이미지">

그래서 한쪽 WSL에서 포트를 열고 한쪽 WSL에서 포트를 받아서 출력해주는 그런 과정으로 풀어야 되는것을 알 수 있습니다

<img src="assets/bandit/20-2.png" width="400" alt="Bandit 19-20 풀이 이미지">

그래서 ./suconnect port번호 로 한쪽에서 열면 한쪽에선 접속한 후 암호를 받을 수 있게됩니다


<img src="assets/bandit/20.png" width="400" alt="Bandit 19-20 풀이 이미지">


 **비밀번호** 가 나와서 bandit20으로 넘어갔습니다.
