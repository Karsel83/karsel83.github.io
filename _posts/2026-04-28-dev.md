## 1. Git 기본 키워드 및 로컬 작업

가장 먼저 로컬 환경에서 작업을 관리할 때 사용하는 명령어들입니다.

- **commit**: 현재 변경 사항을 스냅샷으로 기록합니다. (메시지는 항상 명확하게!)
- **branch**: 독립적인 작업 공간을 분리합니다. 새로운 기능을 개발할 때 필수입니다.
- **checkout / switch**: 작업 중인 브랜치를 전환하거나 특정 커밋으로 이동합니다.
- **merge**: 두 브랜치의 이력을 하나로 합칩니다.
- **rebase**: 브랜치의 기반(Base)을 다른 커밋으로 변경하여 커밋 히스토리를 한 줄로 깔끔하게 유지합니다.
- **cherry-pick**: 다른 브랜치에 있는 특정 커밋 하나만 쏙 골라 현재 브랜치에 적용합니다.

---

## 2. 자유자재로 이동하기: 상대 참조

Git에서 커밋을 일일이 해시값으로 찾는 것은 번거롭습니다. 이때 유용한 것이 **상대 참조**입니다.

- **상대참조 #1 (`^`)**: 한 단계 위의 부모 커밋을 가리킵니다. (예: `main^`)
- **상대참조 #2 (`~`)**: 여러 단계를 한 번에 올라갈 때 사용합니다. (예: `main~3`은 3단계 위 커밋)

---

## 3. 원격 저장소와 협업 (Remote Operations)

GitHub와 같은 원격 저장소와 데이터를 주고받는 핵심 명령어입니다.

- **fetch**: 원격 저장소의 최신 이력을 로컬로 가져오기만 합니다. (내 코드에 바로 합쳐지지 않음)
- **pull**: `fetch`와 `merge`를 동시에 수행합니다.
- **push**: 로컬의 커밋들을 원격 저장소에 업로드합니다.
- **clone**: 원격 저장소의 프로젝트를 통째로 내 컴퓨터로 복사해옵니다.
- **pull --rebase**: 원격의 내용을 가져올 때 `merge` 대신 `rebase`를 사용하여 히스토리를 깔끔하게 관리합니다. 협업 시 자주 권장되는 방식입니다.
- **fakeTeamwork**: (학습용 환경에서 주로 사용) 다른 사람이 커밋을 한 것처럼 시뮬레이션하여 협업 상황을 연습합니다.

---

## 4. 고급 활용: 인자 확장판

단순히 `git push`만 하는 것이 아니라, 특정 브랜치를 지정하거나 이름을 바꿔서 보낼 수 있습니다.

- **push origin [Source] : [Destination]**
  - 로컬의 `[Source]` 브랜치를 원격의 `[Destination]` 브랜치로 보냅니다. 이름이 다를 때 유용합니다.
- **push origin [Source]^ : [Destination]**
  - 로컬 `[Source]`의 부모 커밋을 원격의 특정 위치로 보낼 때 사용합니다.
- **fetch / pull 도 인자 사용 가능**: 원격의 특정 브랜치만 골라서 가져올 때 같은 문법을 사용합니다.
