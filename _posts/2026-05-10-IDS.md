# 침입 탐지 및 방지 시스템 (IDS/IPS) 가이드

네트워크 보안의 핵심은 단순히 **외부 접근을 막는 것을 넘어, 누가 어떤 방식으로 침입하려 하는지 탐지하고 즉각 대응하는 것**입니다.
현대 기업 및 기관 환경에서는 방화벽(Firewall)만으로는 다양한 공격을 방어하기 어렵기 때문에, IDS와 IPS가 필수적인 보안 체계로 자리 잡았습니다.

본 문서에서는 IDS/IPS의 개념부터 동작 원리, 탐지 기법, 구조, 실제 공격 사례, 오픈소스 도구, 우회 기법, 최신 보안 트렌드까지 실무 중심으로 자세히 정리합니다.

---

## 1. IDS와 IPS의 정의 및 개요

네트워크 보안 솔루션인 IDS와 IPS는 **맬웨어, 무단 액세스, DoS 공격 등으로부터 네트워크를 보호하는 중요한 역할**을 수행합니다.

### IDS (Intrusion Detection System, 침입 탐지 시스템)
*   **정의:** 네트워크나 시스템에서 발생하는 비정상적이거나 악의적인 활동을 탐지하고 관리자에게 경고(Alert)를 보내는 시스템입니다.
*   **특징:**
    1) 공격을 직접 차단하지 않음  
    2) 로그 기록 및 경고 생성 중심  
    3) 수동적(Passive) 보안 시스템  
    4) 실시간 모니터링 수행  
    
*   **역할:**
    1) 비정상 로그인 탐지  
    2) 포트 스캔 탐지  
    3) DDoS 징후 탐지  
    4) 악성코드 통신 식별  
    5) 내부자 위협 탐지  
    
비유를 하자면 건물 내부를 감시하며 이상상황이 발생시 경보를 울리는 CCTV와 유사합니다.

### IPS (Intrusion Prevention System, 침입 방지 시스템)
*   **정의:** IDS의 탐지 기능에 더해, 탐지된 위협을 실시간으로 차단하거나 연결을 종료하는 보안 시스템입니다.
*   **특징:**
    1) 공격 패킷 차단  
    2) 세션 종료(TCP Reset)  
    3) IP 차단  
    4) 방화벽 정책 자동 수정 가능  
    5) 능동적(Active) 보안 시스템  
    
*   **역할:**
    1) 악성 패킷 삭제  
    2) 봇넷 통신 차단  
    3) 익스플로잇 방어  
    4) 웹 공격 차단  
    5) 악성 행위 세션 종료  
    
비유를 하자면 출입구에서 신원을 검사하고 위험 인물의 출입을 직접 막는 경비원과 유사합니다.

## 2. IDS vs IPS 주요 차이점 비교

두 시스템은 목적은 비슷하지만 작동 방식과 네트워크상 위치에서 큰 차이를 보입니다.

| 구분 | IDS (침입 탐지) | IPS (침입 방지 + 차단) |
| :--- | :--- | :--- |
| **작동 방식** | 수동적 모니터링 (Passive) | 능동적 대응 (Active) |
| **주요 목적** | 탐지 및 알림 (Alert) | 탐지 및 즉각 차단 (Drop/Reset) |
| **네트워크 위치** | 대역 외 (Out-of-path, Mirroring) | 인라인 (In-line, 길목 위치) |
| **트래픽 차단** | 불가능 | 가능 |
| **성능 영향** | 비교적 낮음 | 상대적으로 높음 (패킷 전수 검사) |
| **장애 발생시** | 탐지 기능만 중단 | 전체 통신 장애 가능 |
| **위험 요소** | 경고 누락 가능 | 오탐 시 정상 서비스 차단 가능 |
| **실패 시 영향** | 보안 공백 발생 (장비 장애 시 트래픽은 흐름) | 네트워크 단절 가능성 (장비 장애 시 통신 불가 가능성) |

## 4. 동작 과정: 어떻게 위협을 탐지하는가?

### ① 패킷 수집 
*  네트워크 인터페이스로 패킷을 캡쳐
*  TCP/IP 헤더 분석
*  Payload 분석
이때 사용하는 기술로는 libpcap, WinPcap, DPDK가 있음

### ② 전처리 
*   패킷을 재조립
*   TCP Stream을 재구성
*   인코딩 해석
이 과정이 중요한 이유 : 공격자는 패킷 분할 기법으로 탐지를 우회할 수 있기 때문임

### ③ 탐지 엔진 분석
*   탐지 엔진이 시그니처 또는 이상 여부를 검사합니다.
분석 대상 : URL/ HTTP Header/ Shellcode / SQL Query/ Binary Payload등

### ④ 대응

IDS는 네트워크 흐름을 방해하지 않는 범위 내에서 사후 분석과 가시성 확보를 목적으로 동작합니다.

**로그 저장 (Logging):** 탐지된 이벤트의 출발지/목적지 IP, 포트, 페이로드 본문, 탐지 시간 등을 상세히 기록하여 향후 침해 사고 분석(Forensic)의 기초 자료로 활용합니다.
*   **관리자 경고 (Alerting):** 대시보드 알림, 이메일, SMS 등을 통해 보안 담당자에게 즉각적인 위협 발생 사실을 공지합니다.
*   **SIEM 전달:** 로그를 중앙 집중형 통합 보안 관리 시스템(SIEM)으로 전송하여, 다른 보안 장비(방화벽, 서버 로그 등)의 데이터와 상관 분석을 수행할 수 있도록 합니다.
*   **TCP Reset 전송 (제한적):** 수동적 방식임에도 불구하고, 탐지 시 공격자와 서버 양측에 TCP Reset 패킷을 보내 일시적으로 연결을 끊는 능동적 시도를 하기도 합니다.

IPS는 위협이 내부망으로 유입되기 전에 실시간으로 물리적/논리적 차단을 수행합니다.

*   **패킷 삭제 (Packet Drop):** 악성으로 판단된 해당 패킷을 네트워크 상에서 즉시 폐기하여 목적지 서버에 도달하지 못하게 합니다.
*   **연결 종료 (Connection Termination):** 비정상적인 세션을 강제로 종료시키고, 관련 TCP 세션 테이블을 삭제하여 추가적인 통신을 차단합니다.
*   **IP 차단 (Source Blocking):** 공격을 시도한 유해 IP 주소를 방화벽 정책에 동적으로 반영하거나 자체 블랙리스트에 등록하여 일정 기간 동안 모든 접근을 거부합니다.
*   **세션 격리 (Session Isolation):** 의심스러운 세션을 별도의 검증 구역(Quarantine)으로 격리하거나 트래픽 속도(Rate Limiting)를 제한하여 피해 확산을 방지합니다.

## 5. 주요 탐지기법
| 기법 | 원리 | 장점 | 단점 |
| **패턴 기반 (Signature)** | 이미 알려진 공격 DB와 비교 | 오탐이 적고 정확함 | 제로데이 공격 탐지 불가 |
| **이상 탐지 (Anomaly)** | 정상 범주(Baseline) 이탈 감지 | 미알려진 공격 탐지 가능 | 오탐(False Positive) 발생 높음 |
| **행위 분석 (Behavior)** | 프로세스/파일의 비정상 행위 추적 | APT 공격 대응에 유리 | 고도의 분석 알고리즘 필요 |
| **상태 분석 (Stateful)** | 프로토콜 규약 준수 여부 확인 | 프로토콜 비정상 이용 탐지 | 시스템 부하 발생 가능 |

## 6. 탐지 가능한 주요 공격 유형

### 네트워크 공격
* **Port Scanning:** 공격 대상의 어떤 포트가 열려 있는지 확인하는 사전 탐색 행위. (예: Nmap 스캔)
*  **SYN Flood:** TCP 3-Way Handshake 과정의 허점을 이용해 가짜 연결 요청을 대량으로 보내 서버 자원을 고갈시킴.
* **ICMP Flood:** 대량의 Ping 패킷을 보내 네트워크 대역폭을 점유하고 시스템을 마비시킴.
* **ARP Spoofing:** 로컬 네트워크에서 공격자의 MAC 주소를 게이트웨이 주소로 속여 트래픽을 가로챔.
* **DNS Spoofing:** DNS 응답을 조작하여 사용자를 악성 사이트로 유도함.

### 웹 공격
* **SQL Injection:** 입력창에 SQL 쿼리를 주입하여 DB 데이터를 무단 열람하거나 조작함.
* **XSS (Cross-Site Scripting):** 웹 페이지에 악성 스크립트를 삽입하여 방문자의 쿠키를 탈취하거나 세션을 가로챔.
* **CSRF (Cross-Site Request Forgery):** 사용자가 의도하지 않은 요청(비밀번호 변경 등)을 강제로 실행하게 함.
* **File Inclusion (LFI/RFI):** 서버의 파일을 읽거나(Local) 외부의 악성 파일을 실행(Remote)하게 함.
* **Command Injection:** 웹 애플리케이션을 통해 OS 명령어를 직접 실행하여 서버 권한을 장악함.

### 시스템 공격
*  **BOF (Buffer Overflow):** 메모리 버퍼의 크기를 초과하는 데이터를 입력해 프로그램 흐름을 바꾸고 악성 코드를 실행함.
* **Privilege Escalation:** 일반 사용자 권한을 관리자(Root/Admin) 권한으로 격상시키는 공격.
* **Malware Injection:** 실행 중인 정상 프로세스 메모리에 악성 코드를 삽입하여 실행함.
* **Rootkit 설치:** 시스템 깊숙이 숨어 관리자 권한을 유지하고 공격 흔적을 지우는 도구 세트.

### 악성코드 탐지
* **랜섬웨어 (Ransomware):** 파일을 암호화하고 복구를 조건으로 금전을 요구하는 행위 탐지.
* **봇넷 통신 (Botnet):** 감염된 좀비 PC가 공격자의 명령을 받기 위해 대기하는 통신 식별.
* **백도어 연결 (Backdoor):** 보안을 우회하여 시스템에 지속적으로 접근하기 위해 열어둔 뒷문 통신 탐지.
* **C2 (Command & Control) 통신:** 외부 공격자 서버와 데이터를 주고받거나 명령을 수신하는 연결 분석.

## 7. 설치 위치 및 환경에 따른 유형

### 네트워크 기반 (NIDS/NIPS)
*   네트워크 세그먼트의 입구(라우터나 방화벽 뒤)에 위치합니다. 주로 설치 위치는 Router뒤, Firewall 뒤 Core Switch 근처입니다.
*   **장점:** 전체 네트워크 트래픽을 한곳에서 감시하며, 시스템 자원에 영향을 주지 않습니다.
*   **단점:** 암호화된 트래픽 내부를 분석하기 어렵고, 고속 네트워크에서 성능 병목이 생길 수 있습니다.

### 호스트 기반 (HIDS/HIPS)
*   개별 서버나 PC(엔드포인트)에 소프트웨어 형태로 설치됩니다.
*   **장점:** 운영체제 로그, 파일 무결성(Tripwire 등), 프로세스 행위를 직접 감시합니다.
*   **단점:** 모든 기기에 설치해야 하므로 관리 부하가 크고, 해당 기기의 자원을 사용합니다.

## 8. IDS/IPS 우회 기법
공격자는 보안장비의 감시를 피하기 위해 다양한 기법을 사용합니다. 이를 이해해야 역으로 방어 전략을 세울 수 있습니다.

- 패킷 분할 (Fragmentation): 공격 페이로드를 아주 작은 조각으로 나누어 전송하여, 재조립 기능이 약한 IDS가 전체 공격 문장을 인식하지 못하게 합니다.
- 인코딩 및 난독화 (Encoding/Obfuscation): UNION SELECT를 %55%4E%49%4F%4E와 같이 URL 인코딩하거나 Base64로 변환하여 시그니처 매칭을 방해합니다.
- 세션 타임아웃 유도: 공격 패킷 사이에 아주 긴 지연 시간을 두어 보안 장비가 세션을 종료하도록 유도한 뒤 나머지를 보냅니다.
- 다형성 악성코드 (Polymorphic Malware): 코드의 기능은 유지하되 외형(해시값 등)을 계속 바꿔 패턴 기반 탐지를 무력화합니다.

## 9. 탐지오류 :  False Positive VS False Negative
보안 장비 운영의 가장 큰 숙제는 이 두 가지 사이의 균형(Trade-off)을 맞추는 것입니다.
## True Positive:
실제 공격을 공격으로 정확히 판단합니다.

## False Negative:
공격이 발생했는데 탐지하지 못함. 보안 사고로 이어지는 가장 위험한 상태입니다.

## False Positive: 
정상적인 활동을 공격으로 오인하여 경보를 울림. 과도한 오탐은 보안 관제사의 피로도를 높이고, IPS의 경우 정상 서비스를 차단하는 대형 사고를 유발합니다.

## 10. 성능 최적화 및 하이엔드 기술
DPI(Deep Packet Inspection)는 많은 자원을 소모하므로 성능 최적화가 필수입니다.

- 하드웨어 가속: FPGA나 ASIC 칩을 사용하여 패킷 분석 속도를 물리적으로 끌어올립니다.
- 멀티코어 분산 처리: 트래픽 로드를 여러 CPU 코어에 분산하여 병목 현상을 방지합니다.
- DPI 최적화: 모든 패킷을 전수 조사하는 대신, 신뢰할 수 있는 세션은 바이패스(Bypass)하는 기술을 적용합니다.

---

## 11. 암호화 트래픽 문제
최근 인터넷 트래픽의 90% 이상이 암호화되어 있어, IPS가 내부를 보지 못하는 '깜깜이' 현상이 발생합니다.

### 해결 방법
- SSL Inspection: 가시성 장비(SSLVA)를 통해 암호화된 트래픽을 복호화하여 IPS에 전달하고, 검사 후 다시 암호화하여 보냅니다.
- JA3 Fingerprinting: 패킷을 복호화하지 않고도 클라이언트와 서버 간의 TLS 핸드셰이크 특징만으로 악성 봇이나 도구를 식별하는 기술입니다.

| :--- | :--- | :--- |
| **오픈소스** | **Snort** | 전 세계 표준과 다름없는 도구. 커뮤니티 룰이 매우 방대함. |
|  | **Suricata** | 멀티스레딩 지원으로 고속 네트워크 환경에 최적화됨. |
|  | **Zeek (Bro)** | 분석 및 포렌식 로그 생성에 특화되어 침해 사고 분석에 유리. |
| **상용 (HW)** | **Palo Alto** | 차세대 방화벽(NGFW) 선두주자. 애플리케이션 식별(App-ID)이 강력함. |
| | **Fortinet** | 자체 ASIC 칩 사용으로 가성비 대비 처리 성능이 매우 우수함. |
|  | **Cisco** | 기존 네트워크 인프라와의 통합 및 위협 인텔리전스 업데이트가 빠름. |

---

## 13. 방화벽(Firewall) vs IDS/IPS 상세 비교 (13번 항목)

| 비교 항목 | 방화벽 (Firewall) | IDS / IPS |
| :--- | :--- | :--- |
| **주요 계층** | L3 (Network), L4 (Transport) | L7 (Application) 중심 |
| **분석 단위** | IP 주소, 포트(Port), 프로토콜 | 페이로드(Payload), 데이터 본문, 행위 |
| **검사 방식** | 패킷 헤더 중심 (ACL 기반 제어) | 깊은 패킷 검사 (DPI) |
| **핵심 역할** | 비인가된 접근의 원천 차단 | 허용된 통신 내 숨겨진 악성 공격 탐지 |
| **비유** | **성문의 문지기:** 신분증(IP) 확인 | **검문소 수색대:** 가방(데이터) 속 흉기 확인 |

---

## 14. 실무 활용 사례: Snort (오픈 소스 IDS/IPS)

보안 실무에서 가장 유명한 툴인 **Snort**는 다음과 같은 모드를 지원합니다.

*   **패킷 스니퍼:** 네트워크의 TCP/IP 패킷을 실시간으로 읽어 보여줌.
*   **패킷 로거:** 트래픽을 저장하여 사후 디버깅 및 분석에 활용.
*   **NIDS 모드:** 설정된 규칙에 따라 공격 시도를 탐지.
*   **Snort Inline:** 탐지와 동시에 공격을 차단하는 IPS 기능 수행.

`# 룰 헤더 (Action / Protocol / Address / Port / Direction)

alert tcp any any -> 192.168.1.10 80 (

    # 룰 옵션
    
    msg:"SQL Injection - UNION detected"; # 탐지 시 출력될 메시지
    
    content:"UNION SELECT";              # 찾을 문자열 (페이로드)
    
    nocase;                              # 대소문자 구분 없음
    
    offset:10;                           # 10바이트 이후부터 검색
    
    sid:1000001;                         # 룰 식별 번호
    
    rev:1;                               # 룰 버전
)`

## 15. 장단점 및 도입 시 고려사항

### 이점
1.  **위협 탐지 향상:** 실시간 모니터링을 통한 조기 대응.
2.  **규제 준수:** ISMS, PCI DSS, GDPR 등 컴플라이언스 요구 사항 충족.
3.  **가시성 확보:** 네트워크 트래픽 패턴에 대한 인사이트 제공.

### 단점 및 한계
1.  **오탐 관리:** 정상 트래픽을 공격으로 오해하여 차단할 경우 서비스 장애 발생(IPS의 경우 치명적).
2.  **암호화 트래픽:** HTTPS 등 암호화된 데이터 속의 공격은 복호화 과정 없이는 탐지 불가.
3.  **지속적 관리:** 매일 발생하는 새로운 취약점에 대해 시그니처를 업데이트해야 함.

## 16. 결론: 다층 보안 전략의 필요성

단일 솔루션만으로는 완벽한 보안이 불가능합니다.
1.  **방화벽**으로 1차 접근 제어
2.  **IPS**로 외부 악성 공격 실시간 차단
3.  **IDS/HIDS**로 내부 이상 징후 및 파일 변조 감시
4.  **SIEM**으로 로그를 통합하여 AI 기반 분석

이러한 **다층 방어(Defense in Depth)** 체계를 구축하는 것이 사이버 위협으로부터 자산을 보호하는 가장 현명한 방법입니다.
