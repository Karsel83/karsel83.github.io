## 1. 퍼징(Fuzzing) 개요
퍼징은 소프트웨어 취약점을 탐색하기 위해 대상 프로그램에 변조된 데이터를 지속적으로 입력하여 비정상적인 종료(Crash)를 유도하는 자동화된 보안 테스트 기법입니다.

### 수행 방식에 따른 분류
* **Black-box Fuzzing**: 프로그램 내부 구조를 모르는 상태에서 입력을 생성합니다. 구현이 쉽지만 효율성이 낮습니다.
* **White-box Fuzzing**: 소스 코드를 완전히 분석하여 모든 경로를 탐색합니다. 정확도는 높으나 분석 오버헤드가 매우 큽니다.
* **Grey-box Fuzzing**: 프로그램 소스 코드의 컴파일 단계에서 계측(Instrumentation)을 추가하여 실행 경로에 대한 피드백을 받습니다. 속도와 효율성의 균형이 좋아 가장 널리 사용됩니다.

## 2. 주요 퍼저(Fuzzer) 분석

| 퍼저명 | 주요 특징 및 차이점 | 비고 |
| :--- | :--- | :--- |
| **AFL++** | Grey-box 퍼저의 대표 주자로, 다양한 변이 알고리즘과 높은 성능을 제공합니다. | 현재 실습 도구 |
| **Jackalope** | Windows 및 macOS 타겟팅이 가능하며 커스텀이 용이한 퍼저입니다. | - |
| **LibFuzzer** | 타겟 함수를 직접 호출하는 In-process 퍼징 방식으로 매우 빠른 속도를 가집니다. | Google Fuzzing 도구 |

## 3. 실습 진행: Exercise 1 (Xpdf)

### 환경 구성 및 빌드
Xpdf 3.02 버전을 대상으로 하며, AFL++ 전용 컴파일러를 사용하여 실행 경로를 추적할 수 있도록 빌드했습니다.

```bash
# AFL++ 컴파일러 설정 및 빌드
CC=afl-clang-fast CXX=afl-clang-fast++ ./configure --prefix="$(pwd)/../install/"
make
make install
