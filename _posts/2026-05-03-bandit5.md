## 3. bandit 22->23 문제
풀이 : 이번 문제는앞전 문제와 유사하게 cron.d로 이동하고 파일확인한뒤 다음 단계로 가기위한
bandit23의 파일을 cat해서 알아보니 bandit23.sh이 나와서 

<img src="assets/bandit/22-1.png" width="400" alt="Bandit 22-23 풀이 이미지">

이 bandit23.sh도 cat해서 뭐가 들어있는지 확인을 해보니,

여기도 앞전 문제처럼 하면 되겠다 해서 echo [password] | nc localhost 를 썼더니 실패했습니다.
그래서 저 문장이 무슨 의미인지 알아보면,
myname=$(whoami)
현재 이 스크립트를 실행 중인 사용자의 이름을 myname 변수에 저장합니다.

mytarget=$(echo I am user $myname | md5sum | cut -d ' ' -f 1)
이 부분이 핵심입니다. "I am user [사용자명]"이라는 문장을 MD5 해시 알고리즘으로 암호화하여 복잡한 문자열을 만들고, 
이를 mytarget 파일 이름으로 사용합니다.

cat /etc/bandit_pass/$myname > /tmp/$mytarget
해당 사용자의 비밀번호 파일 내용을 읽어서, 위에서 생성한 임시 파일( /tmp/[해시값])에 저장합니다.

그래서 저 핵심인 부분을 echo "i an user bandit23" | ...해서 
알아내면 $mytarget이라는 부분이 출력됩니다.

<img src="assets/bandit/22-2.png" width="400" alt="Bandit 22-23 풀이 이미지">

이걸 저 문장에 맞게 cat /tmp/$mytarget하면

<img src="assets/bandit/22-3.png" width="400" alt="Bandit 22-23 풀이 이미지">

**비밀번호** 가 나와서 bandit23으로 넘어갔습니다.

## 4. bandit 23->24문제
풀이: 이번문제도 앞전 문제와 비슷하게 ls-al하고 cron.d에 들어갔습니다.
여기 문제에선 직접 쉘 스크립트를 짜서 카피한다음 그걸 cat으로 읽고 내가 만든 파일로 다시 돌아가면
그 파일에 비밀번호가 생기는 그런 시스템인데,
1. cp가 안됨 2.vi로 쓰는 셸 스크립트의 오류로 인해서 못만들었고

<img src="assets/bandit/23-1.png" width="400" alt="Bandit 23-24 풀이 이미지">

블로그 검색도하고 활용법을 익힌다음에 다시 도전했습니다.

일단 제가 만들었던 my24.sh라는 파일을 만들어서 스크립트 내용으로,

#!/bin/bash
cat /etc/bandit_pass/bandit24 > /tmp/mydir333/rst.txt
를 넣어서 실행한다면 rst.txt에 24로가는 비밀번호를 저장하도록 하는것입니다.

<img src="assets/bandit/231.png" width="400" alt="Bandit 23-24 풀이 이미지">

chmod 777 /tmp/mydir333/my24.sh를 통해 크론잡(시스템)이 이 파일을 실행할 수 있게 만듭니다.

chmod o+w /tmp/mydir333를 시도하여 시스템이 결과물인
rst.txt를 생성할 수 있도록 디렉토리 문을 열어줍니다.

<img src="assets/bandit/232.png" width="400" alt="Bandit 23-24 풀이 이미지">

작성한 스크립트를 /var/spool/bandit24/ 디렉토리로 복사하려고 시도합니다.

<img src="assets/bandit/233.png" width="400" alt="Bandit 23-24 풀이 이미지">

그래서 최종적으로 rst.txt를 cat해서보면 **비밀번호** 가 나와서 bandit24으로 넘어갔습니다.

## 5. bandit 24->25문제
풀이: 대량의 비밀번호 조합을 서버에 한꺼번에 보내는 
**브루트포싱(Brute-forcing)** 능력을 테스트하는 과정입니다.

이 문제를 일일이 10000개를 입력할 수 없어서 쉘 스크립트로 먼저 작성을 해보면

password24="gb8KRRCsshuZXlOtUuR6ypOfjiZbf3G8"

for i in {0000..9999}
do
    echo "$password24 $i">> passlist.txt
done | nc localhost 30002

이제 이 스크립트를 mkdir로 ml25에 directory를 만들고 test.sh에 vi편집기로 넣습니다

<img src="assets/bandit/24-1.png" width="400" alt="Bandit 24-25 풀이 이미지">

그런다음 chmod로 권한을 부여한뒤 우리가 실행을 했을때 passlist.txt로 10000개의 입력을한 결과를
받게 됩니다.

<img src="assets/bandit/24-2.png" width="400" alt="Bandit 24-25 풀이 이미지">

그래서 cat passlist.txt | nc localhost 30002로 입력을 하게 되면,

<img src="assets/bandit/24-3.png" width="400" alt="Bandit 24-25 풀이 이미지">

bandit25의 비밀번호를 받게 되었습니다.
